# 固定问题文档模板

默认问题归档文档：

- `docs/problem-investigation-log.md`

如果项目已有固定问题文档，继续追加到已有文档，不要创建第二份主记录。

## 使用规则

1. 每定位出一个新问题，追加一个新条目。
2. 未确认根因时，状态只能写为 `待定位` 或 `已定位待验证`。
3. 修复后必须补充验证方法和结论。
4. 关闭前必须补“下次遇到时优先检查”。

## 模板

```md
## [YYYY-MM-DD-序号] 问题标题

- 日期：
- 状态：`待定位 / 已定位 / 已修复 / 已验证 / 待回归`
- 现象：
- 预期行为：
- 触发步骤：
- 影响范围：
- 关联模块：
- 排查链路：
- 第一轮日志位置：
- 第二轮日志位置：
- 关键日志特征：
- 根因：
- 修复方式：
- 验证方法：
- 结论：
- 下次遇到时优先检查：
- 相关文件：
- 备注：
```

## 强制字段

后续复查最有价值的字段，必须填写：

- `排查链路`
- `关键日志特征`
- `根因`
- `下次遇到时优先检查`

## 填写建议

### 排查链路

不要写成泛泛描述，直接写实际收敛路径，例如：

- `入口 -> 状态判断 -> 外部调用 -> 回调 -> UI 更新`

### 关键日志特征

写最能直接判断问题的日志，不要把整段日志原样搬进去。

### 根因

必须写确认结论，不写猜测句式。

### 下次遇到时优先检查

写未来复查时最先看的两个到四个检查点，帮助快速复用这次经验。
